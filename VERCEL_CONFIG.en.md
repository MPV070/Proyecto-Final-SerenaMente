# Vercel Configuration for 3 Environments

## 🔴 DEVELOPMENT (dev.serenamente.miniserver.online)

```
Environment Variables:
  NODE_ENV=development
  DOMAIN=dev.serenamente.miniserver.online
  VERCEL_ENV=development
  
Database:
  Name: serena_mente_dev
  Host: [your-mysql-server]
  User: serena_user
  Password: [secure-password]

Deploy:
  - Trigger: Push to 'development' branch
  - Preview URL: Auto-generated by Vercel
  - Production URL: https://dev.serenamente.miniserver.online
```

## 📋 Steps to Configure in Vercel

1. **Go to Dashboard > Project Settings > Environment Variables**

2. **For each environment (dev, staging, prod), add:**
   - NODE_ENV
   - DOMAIN
   - DB_HOST (optional, if database is in the cloud)
   - DB_USER
   - DB_PASSWORD
   - DB_NAME

3. **Configure Domains:**
   - Settings > Domains
   - Add: dev.serenamente.miniserver.online
   - Add: staging.serenamente.miniserver.online
   - Add: serenamente.miniserver.online

4. **Configure Git Branches:**
   - Production Branch: main
   - Preview Branches: development, staging

5. **Protect Main Branch (recommended):**
   - Require Approval for production deployments
   - Settings > Git > Protected Branches

## 🔄 Deployment Workflow

```
Feature Development:
  1. git checkout -b feature/new-feature
  2. Make changes
  3. git push origin feature/new-feature
  4. Vercel creates Preview URL automatically

Testing in Development:
  1. git push origin development
  2. Automatically deploys to dev.serenamente.miniserver.online
  3. Verify at: https://dev.serenamente.miniserver.online

Testing in Staging:
  1. git push origin staging
  2. Automatically deploys to staging.serenamente.miniserver.online
  3. Verify at: https://staging.serenamente.miniserver.online

Deploy to Production:
  1. git push origin main (requires approval if configured)
  2. Automatically deploys to serenamente.miniserver.online
  3. Verify at: https://serenamente.miniserver.online
```
